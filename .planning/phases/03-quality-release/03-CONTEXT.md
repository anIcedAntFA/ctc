<decisions>
## Implementation Decisions

### Unit Test Structure
- **D-01:** Test files live in `tests/unit/` directory mirroring `src/` structure (e.g., `tests/unit/clipboard/copy.test.ts`). Not co-located — keeps `src/` clean and tests out of dist.
- **D-02:** `navigator.clipboard` mocked via `vi.stubGlobal` at module level in each test file, restored in `afterEach`. Matches `.claude/rules/testing.md` pattern — no shared mock factory.
- **D-03:** 100% line and branch coverage on all core clipboard functions (`copy.ts`, `read.ts`, `detect.ts`, `fallback.ts`, `errors.ts`). Barrel files and type-only files excluded from coverage requirement.
- **D-04:** Coverage enforced via Vitest V8 provider with `coverage.thresholds`. CI fails if coverage drops below 100% on core files.

### E2E Test Scope
- **D-05:** Playwright tests cover: `copyToClipboard` happy path, `readFromClipboard` happy path, `isClipboardSupported` returns true on HTTPS, `copyToClipboardLegacy` copies text. Plus one test confirming functions return `false` on insecure context (HTTP served page).
- **D-06:** Clipboard permissions granted via Playwright `browserContext` options: `grantPermissions: ['clipboard-read', 'clipboard-write']`. Applied in Chromium and WebKit. Firefox handles clipboard differently — test without explicit grant there.
- **D-07:** E2E tests run across Chromium, Firefox, and WebKit. Test file: `tests/e2e/clipboard.spec.ts`.
- **D-08:** E2E tests use `page.evaluate()` to call library functions directly — not via UI interactions. Matches `.claude/rules/testing.md` pattern.

### CI Pipeline
- **D-09:** CI runs on **every PR** — full pipeline: lint → build → unit tests (with coverage) → validate (publint, size-limit, attw) → E2E tests. No split between PR and merge checks.
- **D-10:** OS/Node matrix: `ubuntu-latest` × `[20, 22]` for unit tests and validation. E2E runs on `ubuntu-latest` Node 22 only (Playwright browsers sufficient for cross-browser confidence).
- **D-11:** Playwright browser binaries cached between CI runs using `actions/cache` keyed on Playwright version. Prevents slow re-downloads on every run.
- **D-12:** CI workflow file: `.github/workflows/ci.yml`. Separate release workflow: `.github/workflows/release.yml`.

### README Content
- **D-13:** Full API reference in README — self-contained, no separate docs site for v0.1.0. Sections: Installation, Quick Start, API Reference (each function with signature + example + browser support note), Error Handling, Browser Support table, Contributing.
- **D-14:** Each API function documented with: TypeScript signature, one working code example, return value description, browser support note (Clipboard API vs legacy fallback).
- **D-15:** Badges in README: npm version, bundle size (via bundlephobia), CI status, license.

### Changelog & Versioning
- **D-16:** Changelog generated by `@changesets/changelog-github` — auto-adds GitHub PR links. Already configured in `.changeset/config.json`. No hand-written changelog.
- **D-17:** Semver policy: `0.x.y` pre-1.0 — breaking changes bump minor (`0.1.0 → 0.2.0`), non-breaking bump patch (`0.1.0 → 0.1.1`). Consistent with Phase 1 D-18.

### npm Publish Workflow
- **D-18:** Publish trigger: Changesets Release Action (`changesets/action`). On push to `main` with a `.changeset` file present → opens "Version PR" with bumped version + generated CHANGELOG.md. Merging the Version PR → automated `npm publish`.
- **D-19:** npm publish requires `NPM_TOKEN` secret in GitHub repository settings. Document this in CONTRIBUTING.md or README.
- **D-20:** `GITHUB_TOKEN` secret needed for the Release Action to open PRs. Standard GitHub Actions token — no extra setup.

<code_context>
## Existing Code Insights

### Reusable Assets
- `src/clipboard/copy.ts`, `read.ts`, `detect.ts`, `fallback.ts` — All core functions to be tested. All have explicit return types and typed error paths.
- `src/utils/errors.ts` — `createError()`, `handleError()` — test error routing logic here.
- `src/utils/env.ts` — `isBrowser()`, `isSecureContext()` — mock these in unit tests for SSR/HTTP path coverage.
- `dist/` — Already building ESM + CJS. Playwright E2E tests can import from `dist/` or use `src/` via Vite test server.

### Established Patterns
- **Guard-first functions:** Every function checks `isBrowser()` → API availability → `isSecureContext()`. Unit tests must cover each guard returning `false`.
- **Never-throw contract:** All functions return `boolean`/`null`. Unit tests verify no exceptions escape even with throwing `onError` callbacks (post-fix from Phase 2).
- **Named exports only:** No default exports. Import in tests as `import { copyToClipboard } from '../../src/clipboard/copy.ts'`.

### Integration Points
- Vitest config needs `coverage.include: ['src/**']` and `coverage.exclude: ['src/**/index.ts', 'src/**/*.types.ts']`
- Playwright needs a local HTTPS server or `localhost` with `--ignore-certificate-errors` for secure context tests
- GitHub Actions needs `NPM_TOKEN` secret for publish step and `GITHUB_TOKEN` (auto-provided) for Release Action PR creation

</code_context>
