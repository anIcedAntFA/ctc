    ### Why is esbuild in devDependencies?

    `esbuild` appears in `benchmarks/package.json` devDependencies but is **not** the library's build tool — that is [tsdown](https://github.com/sxzz/tsdown).

    `esbuild` is used exclusively as a **bundle size measurement instrument** inside `benchmarks/scripts/measure-bundle-size.ts`. The script calls `esbuild.buildSync({ bundle: true, minify: true, format: 'esm', platform: 'browser' })` to produce a single minified ESM bundle for each competitor library, then measures the compressed size with Node.js `zlib.gzipSync` and `zlib.brotliCompressSync`. The results feed the Bundle Size Comparison table in `BENCHMARKS.md`.

    **Role summary:**
    - `tsdown` — builds the library output in `dist/` for publication
    - `esbuild` — measures minified competitor bundles for `BENCHMARKS.md` only

    ## Q7. Tại sao `esbuild` có trong `devDependencies` khi đang dùng `tsdown`?

    Khi xem `benchmarks/package.json`, bạn thấy `esbuild` trong `devDependencies` và thắc mắc: "Đã dùng `tsdown` để build rồi, tại sao còn cần `esbuild`?"

    ### Câu trả lời: Hai công cụ, hai vai trò hoàn toàn khác nhau

    | Công cụ | Vai trò | Vị trí trong monorepo |
    |---------|---------|----------------------|
    | `tsdown` | **Build tool** — compile TypeScript, tạo `dist/` cho publish | `packages/*/` |
    | `esbuild` | **Measurement instrument** — đo bundle size competitor | `benchmarks/` only |

    ### `esbuild` được dùng để làm gì chính xác?

    File `benchmarks/scripts/measure-bundle-size.ts` dùng `esbuild.buildSync()` để:

    1. Bundle từng competitor library thành một file ESM minified (tương đương cách user thực tế import)
    2. Nén file đó bằng `zlib.gzipSync()` và `zlib.brotliCompressSync()` để lấy kích thước gzip/brotli
    3. Ghi kết quả vào bảng "Bundle Size Comparison" trong `BENCHMARKS.md`

    ```typescript
    // Đây là measurement, không phải build
    esbuild.buildSync({
      bundle: true,
      minify: true,
      format: 'esm',
      platform: 'browser',
    })
    ```

    **`esbuild` không bao giờ được dùng để build `dist/` của library** — đó là nhiệm vụ của `tsdown`. `esbuild` chỉ nằm trong `benchmarks/package.json` (workspace riêng), không ảnh hưởng đến published packages.

    ### Tại sao không dùng `tsdown` để đo bundle size?

    `tsdown` được tối ưu cho DX khi develop (dual CJS/ESM output, sourcemaps, TypeScript declarations). `esbuild` nhanh hơn và cho ra single-file output phù hợp để đo kích thước chuẩn — đúng cách người dùng thực tế sẽ bundle library vào app của họ.

    **TL;DR:** `esbuild` trong benchmarks = thước đo, không phải build tool. `tsdown` = build tool của library.
