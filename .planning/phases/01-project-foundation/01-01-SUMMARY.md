## Decisions Made
- **tsconfig uses noEmit + allowImportingTsExtensions:** Since tsdown (via Rolldown/Oxc) handles all emit (JS + declarations), tsc is only used for type checking. This allows .ts import extensions required by nodenext module resolution without needing tsc to emit.
- **typesVersions for node10 fallback:** attw checks node10 resolution which doesn't support exports maps. Added typesVersions to provide type resolution for clipboard subpath in legacy environments.
- **Let tsdown own the exports map:** tsdown's exports:true overwrites the exports field on each build. Manual types conditions are stripped. This is acceptable because attw validates resolution across all modes with the current setup.
