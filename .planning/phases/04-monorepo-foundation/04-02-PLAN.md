### Task 1: Update `lefthook.yml` for monorepo paths

**Files:**
- `lefthook.yml` (modified)

**Action:**

The current `lefthook.yml` has four concerns to address:

1. **`typecheck` command:** Currently runs `pnpm tsc --noEmit` from root. In the monorepo, the root `tsconfig.json` covers only root config files (commitlint.config.ts, etc.). The source code lives in `packages/core/`. Update to run turbo's typecheck task — but first, `packages/core/package.json` has a `"typecheck": "tsc --noEmit"` script (added in Plan 04-01). So the pre-commit hook becomes:

   ```yaml
   typecheck:
     run: pnpm turbo run typecheck
   ```

   This runs `tsc --noEmit` inside `packages/core` via Turbo. The `typecheck` task is NOT in `turbo.json` yet — add it. Open `turbo.json` and add:

   ```json
   "typecheck": {
     "inputs": ["src/**", "tsconfig.json", "tsconfig.node.json"],
     "outputs": []
   }
   ```

   Place it alongside the other task definitions in `turbo.json`. This task has no `dependsOn` — typechecking can run without a build.

2. **`test` pre-commit command:** Currently runs `pnpm vitest run --passWithNoTests`. This runs vitest directly. After the restructure, vitest is a `packages/core` devDependency. Update to call it via turbo OR via the workspace filter. Since pre-commit should be fast (only unit tests, no build), use the filter approach to avoid turbo's build dependency:

   ```yaml
   test:
     run: pnpm --filter=./packages/core exec vitest run --passWithNoTests
     glob: "*.{ts,js}"
   ```

   This runs vitest directly inside packages/core without triggering the `build` dependsOn from `turbo.json`'s test task.

3. **`build` pre-push command:** Update to use turbo:

   ```yaml
   build:
     run: pnpm turbo run build
   ```

4. **`test` pre-push command:** Update to use turbo:

   ```yaml
   test:
     run: pnpm turbo run test
   ```

5. **`validate` pre-push command:** Update to use turbo:

   ```yaml
   validate:
     run: pnpm turbo run validate
   ```

6. **`lint` and `format` pre-commit commands:** These pass `{staged_files}` directly to biome. The `biome.json` at root now has updated includes covering `packages/*/src/**`. Lefthook's `{staged_files}` filters to changed files by path, so biome will correctly lint staged files in `packages/core/src/` when they match `*.{ts,js,json}`. No change needed to the `run:` invocation for these two commands — biome resolves upward from the file path to find the root `biome.json`.

Full rewritten `lefthook.yml`:

```yaml
pre-commit:
  parallel: true
  commands:
    lint:
      run: pnpm biome check --no-errors-on-unmatched {staged_files}
      glob: "*.{ts,js,json}"
    format:
      run: pnpm biome format --no-errors-on-unmatched {staged_files}
      glob: "*.{ts,js,json}"
    typecheck:
      run: pnpm turbo run typecheck
    test:
      run: pnpm --filter=./packages/core exec vitest run --passWithNoTests
      glob: "*.{ts,js}"

commit-msg:
  commands:
    commitlint:
      run: pnpm commitlint --edit {1}

pre-push:
  commands:
    build:
      run: pnpm turbo run build
    test:
      run: pnpm turbo run test
    validate:
      run: pnpm turbo run validate
```

Also add the `typecheck` task to `turbo.json` as described above.

**Verify:**
- `cat lefthook.yml` — shows `turbo run typecheck` for pre-commit typecheck
- `cat lefthook.yml` — shows `pnpm --filter=./packages/core exec vitest` for pre-commit test
- `cat lefthook.yml` — shows `turbo run build`, `turbo run test`, `turbo run validate` in pre-push
- `cat turbo.json | grep typecheck` — shows the typecheck task definition

### Task 2: Update `.github/workflows/ci.yml` for Turborepo

**Files:**
- `.github/workflows/ci.yml` (modified)

**Action:**

Replace direct `pnpm lint`, `pnpm build`, `pnpm test`, `pnpm validate` calls with their turbo equivalents. The job structure (lint-and-build → unit-test → e2e-test) stays the same — the dependency gates via `needs:` remain unchanged.

The `on:` trigger currently fires on PRs targeting `main`. The repo uses `master` as the default branch (per release.yml and STATE.md). Update the trigger to `master`:

```yaml
on:
  pull_request:
    branches: [master]
```

(If PRs should target `main`, leave it. But `release.yml` pushes to `master`, and the repo history uses `master`. Change to `master` for consistency.)

Full rewritten `ci.yml`:

```yaml
# .github/workflows/ci.yml
# Runs on every PR targeting master.
# Pipeline: lint-and-build → unit-test (matrix) → e2e-test
# Turbo orchestrates tasks across the monorepo; each job delegates to turbo run.
name: CI

on:
  pull_request:
    branches: [master]

concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

jobs:
  lint-and-build:
    name: Lint & Build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        with:
          version: 10

      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: 'pnpm'

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Lint
        run: pnpm turbo run lint

      - name: Build
        run: pnpm turbo run build

  unit-test:
    name: Unit Tests (Node ${{ matrix.node-version }})
    needs: lint-and-build
    runs-on: ubuntu-latest
    strategy:
      matrix:
        node-version: [20, 22]
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        with:
          version: 10

      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}
          cache: 'pnpm'

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Unit tests with coverage
        run: pnpm turbo run test -- --coverage

      - name: Validate package (publint + attw)
        run: pnpm turbo run validate

  e2e-test:
    name: E2E Tests (Chromium / Firefox / WebKit)
    needs: lint-and-build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        with:
          version: 10

      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: 'pnpm'

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Cache Playwright browsers
        uses: actions/cache@v4
        id: playwright-cache
        with:
          path: ~/.cache/ms-playwright
          key: playwright-${{ hashFiles('**/pnpm-lock.yaml') }}

      - name: Install Playwright browsers (cache miss)
        if: steps.playwright-cache.outputs.cache-hit != 'true'
        run: pnpm exec playwright install --with-deps chromium firefox webkit

      - name: Install Playwright system deps (cache hit)
        if: steps.playwright-cache.outputs.cache-hit == 'true'
        run: pnpm exec playwright install-deps chromium firefox webkit

      - name: Build (needed for E2E fixture to load dist/)
        run: pnpm turbo run build

      - name: E2E tests
        run: pnpm turbo run test:e2e
```

Notes:
- `pnpm turbo run test -- --coverage` passes `--coverage` through turbo to the underlying `vitest run` command. Turbo supports passthrough args via `--`. This works as documented.
- The `pnpm exec playwright install` line stays as-is (not turbo-wrapped) because Playwright browser installation is a setup step, not a task in the pipeline.
- `test:e2e` has `"cache": false` in turbo.json so the E2E step always runs even if inputs are unchanged.
- No turbo remote caching (TURBO_TOKEN) — per REQUIREMENTS.md, remote caching is explicitly out of scope for this project. Local cache only.

**Verify:**
- `cat .github/workflows/ci.yml | grep 'turbo run'` — shows turbo run lint, turbo run build, turbo run test, turbo run validate, turbo run test:e2e
- `cat .github/workflows/ci.yml | grep branches` — shows `[master]`
- `pnpm turbo run test -- --coverage && ls packages/core/coverage/` — exits 0 and coverage/ directory is produced, confirming --coverage flag reaches vitest
- YAML syntax valid: `pnpm exec js-yaml .github/workflows/ci.yml 2>&1` exits 0, OR simply verify no obvious YAML parse errors by inspection
