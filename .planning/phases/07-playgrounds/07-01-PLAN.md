# Phase 7 Plan 01: Workspace foundation + playground/vanilla

## Goal

Wire up the monorepo for playgrounds and scaffold `playground/vanilla` as both an interactive Vite app and the Playwright E2E fixture replacement for `packages/core`.

## Context

- **D-01**: Directory is `playground/` (not `apps/`).
- **D-02**: `playground/vanilla` replaces `packages/core/tests/e2e/fixtures/index.html` as the E2E harness. Playwright `webServer` must serve `playground/vanilla/dist/`. `window.__clipboard` must be preserved.
- **D-03**: Detection panel shows all 4 detect functions evaluated on load.
- **D-05**: Independent styling — vanilla uses inline `<style>` inside `index.html`.
- **D-06**: Copy button, "Copied!" 2s feedback, error code display, secure context badge, detection panel.
- **D-07**: `pnpm-workspace.yaml` adds `"playground/*"`; each playground has `"private": true`.
- **D-08**: `turbo.json` gains `dev: { cache: false, persistent: true }`; root `package.json` gains `"dev": "turbo run dev --filter=./playground/*"`.
- **RESEARCH Pitfall 2**: `window.__clipboard` must be assigned synchronously at the top of `main.ts` before any async code — otherwise `page.waitForFunction` times out.
- **RESEARCH Pitfall 3**: Use compound `webServer.command` that builds playground/vanilla first, so E2E tests are self-contained without a Turbo cross-package dependency.
- **RESEARCH Pitfall 4**: `biome.json` overrides must add `playground/*/*.config.ts` to exempt `noDefaultExport`.
- **RESEARCH E2E wiring**: Navigation changes from `'/tests/e2e/fixtures/'` to `'/'`.

### Task 2: Scaffold playground/vanilla

**Files to create:**
- `playground/vanilla/package.json`
- `playground/vanilla/tsconfig.json`
- `playground/vanilla/vite.config.ts`
- `playground/vanilla/index.html`
- `playground/vanilla/src/main.ts`

**Steps:**

1. Create `playground/vanilla/package.json`:
   ```json
   {
     "name": "@ngockhoi96/playground-vanilla",
     "version": "0.0.0",
     "private": true,
     "type": "module",
     "scripts": {
       "dev": "vite",
       "build": "vite build",
       "preview": "vite preview"
     },
     "dependencies": {
       "@ngockhoi96/ctc": "workspace:*"
     },
     "devDependencies": {
       "typescript": "6.0.2",
       "vite": "8.0.8"
     }
   }
   ```

2. Create `playground/vanilla/tsconfig.json`:
   ```json
   {
     "$schema": "https://json.schemastore.org/tsconfig",
     "extends": "../../tsconfig.base.json",
     "compilerOptions": {
       "noEmit": true,
       "target": "ES2020",
       "lib": ["ES2020", "DOM", "DOM.Iterable"]
     },
     "include": ["src"]
   }
   ```
   The `lib: ["ES2020", "DOM", "DOM.Iterable"]` is mandatory — base tsconfig omits DOM lib but clipboard API needs browser globals.

3. Create `playground/vanilla/vite.config.ts`:
   ```typescript

       outDir: 'dist',
     },
   })
   ```

4. Create `playground/vanilla/index.html` — full interactive UI with inline styles (D-05, D-06, D-03). Render the detection table rows via DOM element creation (not innerHTML) to avoid XSS-flagging by linters — the content is generated from trusted internal data but static analysis cannot verify this:

     <head>

5. Create `playground/vanilla/src/main.ts`. The `window.__clipboard` assignment MUST be at the very top, synchronous, before any async code (Pitfall 2). Use DOM element creation for table rows instead of innerHTML to keep Biome satisfied:
   ```typescript
   import {
     copyToClipboard,
     copyToClipboardLegacy,
     isClipboardApiSupported,
     isClipboardReadSupported,
     isReadPermissionGranted,
     isSecureContext,
     isWritePermissionGranted,
     readFromClipboard,
   } from '@ngockhoi96/ctc'

   // Expose for E2E tests -- synchronous, top-level, unconditional (D-02)
   // Shape must match the Window.__clipboard declaration in clipboard.spec.ts
   ;(
     window as Window & {
       __clipboard: {
         copyToClipboard: typeof copyToClipboard
         readFromClipboard: typeof readFromClipboard
         isClipboardSupported: typeof isClipboardApiSupported
         isClipboardReadSupported: typeof isClipboardReadSupported
         copyToClipboardLegacy: typeof copyToClipboardLegacy
       }
     }
   ).__clipboard = {
     copyToClipboard,
     readFromClipboard,
     isClipboardSupported: isClipboardApiSupported,
     isClipboardReadSupported: isClipboardReadSupported,
     copyToClipboardLegacy,
   }

   // Secure context badge (D-06)
   const secureBadge = document.getElementById('secure-badge')!
   const secure = isSecureContext()
   secureBadge.textContent = secure ? 'Secure Context' : 'Insecure Context'
   secureBadge.classList.add(secure ? 'secure' : 'insecure')

   // Detection panel (D-03) -- use DOM element creation, not innerHTML
   type DetectionRow = { label: string; value: string | boolean | null }
   const detectionRows: DetectionRow[] = [
     { label: 'isClipboardApiSupported()', value: isClipboardApiSupported() },
     { label: 'isSecureContext()', value: isSecureContext() },
     { label: 'isReadPermissionGranted()', value: null },
     { label: 'isWritePermissionGranted()', value: null },
   ]

   const tbody = document.getElementById('detection-tbody')!

   function renderDetection() {
     while (tbody.firstChild) tbody.removeChild(tbody.firstChild)
     for (const row of detectionRows) {
       const tr = document.createElement('tr')
       const tdLabel = document.createElement('td')
       tdLabel.textContent = row.label
       const tdValue = document.createElement('td')
       tdValue.textContent = row.value === null ? 'checking...' : String(row.value)
       tr.appendChild(tdLabel)
       tr.appendChild(tdValue)
       tbody.appendChild(tr)
     }
   }

   renderDetection()

   async function loadPermissions() {
     const [read, write] = await Promise.all([
       isReadPermissionGranted(),
       isWritePermissionGranted(),
     ])
     detectionRows[2]!.value = read
     detectionRows[3]!.value = write
     renderDetection()
   }

   loadPermissions()

   // Copy button interaction (D-06)
   const copyBtn = document.getElementById('copy-btn')!
   const copyInput = document.getElementById('copy-input') as HTMLInputElement
   const feedback = document.getElementById('feedback')!
   let resetTimer: ReturnType<typeof setTimeout> | null = null

   copyBtn.addEventListener('click', async () => {
     const text = copyInput.value
     const success = await copyToClipboard(text, {
       onError: (code) => {
         feedback.textContent = `Error: ${code}`
         feedback.className = 'error'
       },
     })
     if (success) {
       feedback.textContent = 'Copied!'
       feedback.className = 'copied'
       if (resetTimer !== null) clearTimeout(resetTimer)
       resetTimer = setTimeout(() => {
         feedback.textContent = ''
         feedback.className = ''
         resetTimer = null
       }, 2000)
     }
   })
   ```

6. Run `pnpm install` from the repo root to register the new workspace member and symlink `@ngockhoi96/ctc`.

**Verification:**
- `pnpm --filter=@ngockhoi96/playground-vanilla build` exits 0 and `playground/vanilla/dist/index.html` exists.
- `pnpm --filter=@ngockhoi96/playground-vanilla dev` starts without errors (Ctrl+C to stop).
- `pnpm -r lint` passes (no biome errors on playground/vanilla files).

## Threat Model

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Information Disclosure | `window.__clipboard` assignment | accept | Intentional E2E test harness in a private playground. Not published. localhost-only. |
| T-07-02 | Tampering | http-server serving `dist/` | accept | localhost-only; no sensitive data served; `--cors` is standard for local dev. |
| T-07-03 | Denial of Service | compound `webServer.command` build step | accept | Build failures fail CI loudly. Playwright exits non-zero on server startup failure — no silent degradation. |

## Notes

- **Pitfall 2 (window.__clipboard timing):** The `window.__clipboard` assignment must be the very first statement after imports in `main.ts`. Do NOT put it inside an `async` function or after an `await`. The E2E `page.waitForFunction` checks synchronously available state.
- **Pitfall 3 (build before serve):** The compound `webServer.command` approach (`build && http-server`) is used instead of Turbo cross-package `dependsOn`. This rebuilds on every `pnpm test:e2e` run but is self-contained and CI-safe.
- **Pitfall 4 (noDefaultExport):** Both `files.includes` and `overrides` in `biome.json` must be updated. Adding includes without the override causes lint errors on `vite.config.ts`.
- **window.__clipboard shape:** The spec declares `.isClipboardSupported` and `.isClipboardReadSupported` (not `isClipboardApiSupported`/`isClipboardReadSupported`). The `main.ts` maps exported names to the names the spec expects.
- **DOM construction vs innerHTML:** Detection table rows are built via DOM element creation (`createElement`/`appendChild`) to avoid Biome's XSS lint warning. The content is internal/trusted but static analysis cannot verify this.
