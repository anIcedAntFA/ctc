# Phase 7 Plan 04: playground/svelte

## Goal

Scaffold `playground/svelte` as a Vite + Svelte 5 app demonstrating both `copyAction` (Svelte action) and `useCopyToClipboard` (runes) from `@ngockhoi96/ctc-svelte` in a responsive two-panel layout.

## Context

- **D-01**: Located at `playground/svelte/` (per D-01).
- **D-03**: Detection panel shows all 4 detect functions, evaluated on load.
- **D-04**: Svelte playground has a 2-column desktop layout / stacked mobile layout. CSS grid: `grid-template-columns: 1fr 1fr` desktop, `grid-template-columns: 1fr` mobile (`@media (max-width: 767px)`). No tabs — both panels always visible.
- **D-05**: Independent styling — scoped `<style>` in each `.svelte` component.
- **D-06**: Each panel must demonstrate: copy button, "Copied!" 2s feedback, error code display. Secure context badge shared in bottom section.
- **D-07**: `"private": true`, CI-excluded.
- **D-08**: `"dev": "vite"` script.
- **RESEARCH**: `@sveltejs/vite-plugin-svelte@7.0.0` (same version as in `packages/svelte`). Svelte 5.55.3. No `svelte.config.js` needed — plugin reads compiler options inline. Runes (`$state`, `$effect`, `$derived`) auto-detected per component in Svelte 5; no global `compilerOptions.runes` needed.
- **RESEARCH Pitfall 7**: `useCopyToClipboard` uses `$effect` internally — MUST be called at the top level of a `<script>` block inside a `.svelte` file, never inside a callback or module-level `.ts` file.
- **RESEARCH Pitfall 6**: `@ngockhoi96/ctc-svelte` must have a `version` field in its `package.json` for pnpm `workspace:*` to resolve.
- **Wave 2**: Depends on Plan 01. Runs in parallel with Plans 02 and 03 (no file overlap).

6. Create `playground/svelte/src/CopyAction.svelte` — left/top panel demo using `use:copyAction` directive (D-04). Listens to `ctc:copy` and `ctc:error` custom events on the action target to update local state:
   ```svelte
   <script lang="ts">
     import { copyAction } from '@ngockhoi96/ctc-svelte'

     let text = $state('Hello from copyAction')
     let feedback = $state<string | null>(null)
     let feedbackType = $state<'copied' | 'error' | null>(null)
     let resetTimer: ReturnType<typeof setTimeout> | null = null

     function onCopy() {
       feedback = 'Copied!'
       feedbackType = 'copied'
       if (resetTimer !== null) clearTimeout(resetTimer)
       resetTimer = setTimeout(() => {
         feedback = null
         feedbackType = null
         resetTimer = null
       }, 2000)
     }

     function onError(event: CustomEvent<{ code: string }>) {
       feedback = `Error: ${event.detail.code}`
       feedbackType = 'error'
     }
   </script>

   <div class="panel">
     <h2>copyAction (use: directive)</h2>
     <input type="text" bind:value={text} />
     <button
       use:copyAction={{ text }}
       on:ctc:copy={onCopy}
       on:ctc:error={onError}
     >
       Copy
     </button>
     <div class="feedback" class:copied={feedbackType === 'copied'} class:error={feedbackType === 'error'}>
       {feedback ?? ''}
     </div>
   </div>

   <style>
     .panel { padding: 1rem; border: 1px solid #e0e0e0; border-radius: 6px; }
     h2 { font-size: 1rem; margin-bottom: 0.5rem; }

       margin-bottom: 0.5rem;
   ```

8. Create `playground/svelte/src/App.svelte` — root component with 2-col grid layout (D-04), shared detection panel, and secure context badge. Compose `CopyAction` and `CopyRune` components. Evaluate detection functions on mount:
   ```svelte
   <script lang="ts">
     import {
       isClipboardApiSupported,
       isReadPermissionGranted,
       isSecureContext,
       isWritePermissionGranted,
     } from '@ngockhoi96/ctc'
     import { onMount } from 'svelte'
     import CopyAction from './CopyAction.svelte'
     import CopyRune from './CopyRune.svelte'

     type Detection = {
       clipboardApi: boolean | null
       secureContext: boolean | null
       readPermission: boolean | null
       writePermission: boolean | null
     }

     let detection = $state<Detection>({
       clipboardApi: null,
       secureContext: null,
       readPermission: null,
       writePermission: null,
     })

     const secure = $derived(detection.secureContext)

     onMount(async () => {
       detection.clipboardApi = isClipboardApiSupported()
       detection.secureContext = isSecureContext()
       const [read, write] = await Promise.all([
         isReadPermissionGranted(),
         isWritePermissionGranted(),
       ])
       detection.readPermission = read
       detection.writePermission = write
     })

     const detectionRows = [
       { key: 'clipboardApi' as const, label: 'isClipboardApiSupported()' },
       { key: 'secureContext' as const, label: 'isSecureContext()' },
       { key: 'readPermission' as const, label: 'isReadPermissionGranted()' },
       { key: 'writePermission' as const, label: 'isWritePermissionGranted()' },
     ]
   </script>

   <div class="container">
     <h1>@ngockhoi96/ctc -- Svelte Playground</h1>

     <!-- Two-panel grid layout (D-04) -->
     <div class="panels-grid">
       <CopyAction />
       <CopyRune />
     </div>

     <!-- Shared bottom section -->
     <section class="shared">
       <div class="secure-section">
         <h2>Secure Context</h2>
         {#if secure !== null}
           <span class="badge" class:secure={secure} class:insecure={!secure}>
             {secure ? 'Secure Context' : 'Insecure Context'}
           </span>
         {/if}
       </div>

       <div class="detection-section">
         <h2>Detection Panel</h2>
         <table>
           <thead>
             <tr>
               <th>Function</th>
               <th>Result</th>
             </tr>
           </thead>
           <tbody>
             {#each detectionRows as row (row.key)}
               <tr>
                 <td>{row.label}</td>
                 <td>{detection[row.key] === null ? 'checking...' : String(detection[row.key])}</td>
               </tr>
             {/each}
           </tbody>
         </table>
       </div>
     </section>
   </div>

   <style>
     .container {
       font-family: system-ui, sans-serif;
       max-width: 900px;
       margin: 2rem auto;
       padding: 0 1rem;
     }
     h1 { font-size: 1.25rem; margin-bottom: 1.5rem; }

     /* Two-column desktop layout (D-04) */
     .panels-grid {
       display: grid;
       grid-template-columns: 1fr 1fr;
       gap: 1rem;
       margin-bottom: 1.5rem;
     }

     /* Single-column mobile layout (D-04) */
     @media (max-width: 767px) {
       .panels-grid {
         grid-template-columns: 1fr;
       }
     }

     .shared { margin-top: 1rem; }
     .secure-section { margin-bottom: 1.5rem; }
     h2 { font-size: 1rem; margin-bottom: 0.5rem; }

     .badge {
       display: inline-block;
       padding: 0.2rem 0.5rem;
       border-radius: 4px;
       font-size: 0.75rem;
       font-weight: bold;
     }
     .secure { background: #d4edda; color: #155724; }
     .insecure { background: #f8d7da; color: #721c24; }

     table { border-collapse: collapse; width: 100%; font-size: 0.875rem; }
     th, td { text-align: left; padding: 0.3rem 0.5rem; border-bottom: 1px solid #eee; }
     th { background: #f9f9f9; }
   </style>
   ```

## Notes

- **Pitfall 7 (runes in component context):** `useCopyToClipboard` from `/runes` uses `$effect` internally. It MUST be called at the top level of a `<script>` block in a `.svelte` component. NEVER call it inside event handlers, callbacks, or plain `.ts` modules.
- **Pitfall 6 (workspace:* version):** `@ngockhoi96/ctc-svelte` must have a `version` field in its `package.json`. Check before running `pnpm install`.
- **No svelte.config.js needed:** `@sveltejs/vite-plugin-svelte@7` reads config inline. Svelte 5 runes are auto-detected per component — no `compilerOptions: { runes: true }` needed globally.
- **Biome + .svelte files:** Biome excludes `**/*.svelte` (already configured). Only `main.ts` and `vite.config.ts` are linted for the svelte playground. No action needed.
- **copyAction event handling:** `copyAction` dispatches custom events `ctc:copy` and `ctc:error`. In Svelte 5, use `on:ctc:copy` and `on:ctc:error` event directives on the element that has `use:copyAction`. The `ctc:error` event has `detail.code` with the error code string.
- **CopyRune text reactivity:** `useCopyToClipboard(text)` in `CopyRune.svelte` receives the initial text. When the user edits the input, `ctc.copy(text)` is called with the current reactive value. The rune composable's internal state is reactive to the passed copy function call arguments.
- **Wave 2 parallelism:** Plans 02, 03, and 04 are all Wave 2 with no file overlap — all three can be executed in parallel.
